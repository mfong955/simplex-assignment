Key terms — things I was confused about
Residual stream

The running state vector

A single high-dimensional vector (e.g. 120D) that gets updated by each transformer layer. Each layer adds its contribution rather than replacing. One vector exists per token position in a sequence.

Analogy: a quantum state vector that operators act on layer by layer.
Belief state

The model's current probability distribution

The posterior P(hidden state | all past tokens). This is encoded in the residual stream vector — not stored in the weights. Updated with every new token via Bayesian-like updating.

Analogy: a detective's running mental model of the most likely suspect, updated with each new clue.
Hidden states

The unobservable internal states

The discrete states of the process generating the tokens. You never observe them — only the emitted tokens. For Mess3, there are 3 hidden states {A, B, C}. The model infers their probability from token history.

Analogy: a vending machine secretly switching between "generous" and "stingy" modes — you only see what comes out.
Factors

Independent sub-processes running in parallel

Multiple processes generating tokens simultaneously. Their outputs are combined into one token — the model never sees them separately. Paper uses 5 factors. Each gets its own orthogonal subspace in the residual stream.

Analogy: two independent vending machines whose outputs are mixed together before you see them.
Simplex

The shape belief states live in

A probability distribution over N states must be non-negative and sum to 1. This constraint forces belief states to live on an (N−1)-simplex: line segment for 2 states, triangle for 3, tetrahedron for 4.

Company name: named directly after this geometric object — their entire program is built on it.
Latent structure

The hidden organization generating the data

The underlying factored processes that produce tokens. Never directly observable. Tracking it correctly is required for optimal prediction. If residual stream geometry matches predicted simplices, the model is tracking it.

Analogy: a novel's plot and character motivations — never written explicitly, but generates every word.
Orthogonal subspaces

Separate "rooms" in the residual stream

The transformer dedicates a separate set of dimensions to each factor. These sets are orthogonal — at 90° to each other — so changing one doesn't disturb another. This enables surgical interventions.

Analogy: x, y, z axes — moving along x doesn't affect y or z coordinates at all.
Factored vs. joint representation

Linear vs. exponential dimensions

Joint: tracks all factor combinations. Dimensions = product (3⁵=243). Always lossless.
Factored: tracks each factor separately. Dimensions = sum (5×2=10). Lossless only when factors are conditionally independent.

Weights vs. activations

Fixed machinery vs. dynamic values

Weights/parameters: learned during training, frozen forever. Encode the algorithm.
Activations (residual stream): computed fresh per input. Encode the current belief state. The paper studies activations, not weights.

Analogy: pinball machine bumpers (weights) vs. the ball's position (activations).
Conditional independence

The condition for lossless factoring

Two factors are conditionally independent given a token if observing the token lets you update each factor's belief separately without losing information. When this holds, the factored representation is exactly as good as the joint one.

PCA / CEV

How they measure dimensionality

Run model on 1000s of inputs → collect residual stream vectors → find directions of most variance (principal components). CEV(k) = fraction of variance in top k directions. If CEV(10)≈0.95, activations are effectively 10-dimensional.

Key test: does the model use 10 dims (factored prediction) or 242 dims (joint prediction)?
Inductive bias

Preference baked into the architecture

Transformers find factored representations early in training and stay there — even when a higher-dimensional solution would predict better. The factored solution is a wide, easy-to-find basin in the loss landscape. A gift for interpretability.

Analogy: water supercooling — stuck in liquid state (factored) even though ice (joint) is lower energy.
The Mess3 process — what it is and why it's used

Structure

3 hidden states {A, B, C}. Emits tokens from {0, 1, 2}. Defined by parameters α and x. Non-Markovian — predicting well requires tracking full belief state, not just last token.

Why a triangle?

Belief state = (pA, pB, pC) with pA+pB+pC=1. Three non-negative numbers summing to 1 form a triangle (2-simplex). Two numbers determine the third. So belief state varies in exactly 2 dimensions.

Why use it?

Ground truth is analytically computable. Researchers can calculate exactly what belief state the model should have, then check whether the residual stream matches. Complex enough (fractal geometry) to be a real test.

Gradient descent and loss landscape

The loss landscape is a surface over all possible weight configurations — height = prediction error. Gradient descent feels the local slope and steps downhill. It always moves toward lower error but can get stuck in local minima.

The factored basin: wide, shallow, easy to fall into early. Gradient descent finds it within hundreds of steps.
The joint basin: potentially deeper (lower loss), but narrow and hard to reach. Requires sustained gradient pressure over thousands more steps.

Not like MCMC: gradient descent is deterministic and greedy (always downhill). MCMC is random and exploratory (can escape local traps). Gradient descent gets stuck; MCMC wanders.

Analogy: blindfolded on a hilly landscape — you feel the slope and step downhill. You'll find a valley, but not necessarily the deepest one.
Why this matters — the big picture

Interpretability

If factors live in orthogonal subspaces, we can identify which subspace encodes which aspect of the world. We can then "read out" the model's current belief about specific things — e.g. what topic it thinks it's discussing.

Surgical intervention

Orthogonal subspaces mean you can modify one factor without disturbing others. Like editing one variable in a program. This enables targeted control of model behavior — a major safety research goal.

Verification

If the residual stream geometry matches predicted simplices, the model is genuinely tracking the latent structure of its training data — doing something close to optimal Bayesian inference. If it doesn't match, the model is doing something else we don't yet understand.

Open frontier

This paper proves the decomposition exists in small synthetic models. Applying it to large LLMs (where the ground truth isn't known) requires unsupervised discovery methods — exactly what Simplex is building next.

Physics analogies — the ones that helped
Spectroscopy

You see a combined absorption spectrum (tokens). The model discovers it's a superposition of two atomic species (factors). It separates them into orthogonal frequency channels (subspaces) without being told they exist.

Seismology

One seismometer records earthquake + traffic + ocean waves combined. The model learns to separate the three independent sources into orthogonal channels purely from the statistical structure of the combined signal.

Supercooling

Water below 0°C stays liquid (factored representation) even though ice (joint representation) is lower energy. Needs a push to escape. The factored basin is the metastable state; more training = more pressure to escape toward joint.

Bayesian detective

After each new clue (token), the detective updates their distribution over suspects (hidden states). The current belief = posterior over all evidence so far. It's a sufficient statistic — all useful past information is compressed into it.

The assignment in one sentence per task

Task 1 — non-ergodic dataset

Build training data from multiple Mess3 processes where each sequence comes from only one process. Models language's multi-source reality (legal docs vs. poems). Train a transformer on it.

Task 2 — pre-registered prediction

Before running the experiment, mathematically derive what geometry you expect to see in the residual stream. At short context: mixed/uncertain. At long context: collapsed to one process's simplex.

Task 3 — analyze geometry

After training, run PCA on residual streams. Does dimensionality match factored prediction? Do the geometric shapes match the predicted Mess3 simplices? Does context length affect disambiguation?
