Transformer mechanics — how it actually works

Cheat sheet covering parallel processing, residual streams, attention, activations, and PCA analysis

Core concepts — things I was confused about
Parallel but not independent

All tokens processed at once — but with different information

All 16 tokens enter the transformer simultaneously in one forward pass. But causal masking enforces an information asymmetry: position 7 can only attend to positions 0–7, while position 15 can attend to all 16. So even though computation is parallel, each position has different available context.

Analogy: 16 people working at the same time, but person 7 can only read notes from people 0–7. They finish simultaneously but with different amounts of information.
Final position

The last token in the sequence — not the last layer

"Final position" means token position 15 (in a 16-token sequence) — the position that has seen all prior tokens via attention. It has the most context and produces the most information-rich residual stream vector. This is why PCA plots labeled "final position" use these vectors.

Not to be confused with final layer. You can analyze any combination of (layer, position). The paper often fixes the final layer and varies something else (position, training step).
Residual stream

One vector per position that accumulates across layers

Each token position has a single 128D vector that travels vertically through the network. Each layer adds its output to this vector rather than replacing it. The residual stream at any point = embedding + all layer contributions so far.

Analogy: a whiteboard each layer writes on without erasing. The final residual stream is the whiteboard after all layers have written.
Activations

Any internal values during a forward pass

Broad term for all intermediate numerical values inside the network. The residual stream is the most important activation — it's the running state vector. Other activations (attention head outputs, MLP outputs) are intermediate computations that feed into the residual stream but are not the stream itself.

In this paper: "activations" and "residual stream" are used nearly interchangeably — both refer to the residual stream vector at the final layer.
Attention — what "communication" actually means

Content-dependent, weighted information routing

Not simple addition. Each position computes three things from its current vector via learned matrices:

Query (Q)"What am I looking for?"
Key (K)"What do I contain?"
Value (V)"What do I actually send?"

Q·K dot products → softmax weights → weighted sum of V vectors → added to residual stream. The weights are input-dependent and learned — different inputs produce different attention patterns.

Why attention is not simple addition

Three key differences from just summing vectors

1. Weights are learned and content-dependent — relevant positions get higher weight.

2. Weights vary by input — the same position attends differently to different sequences.

3. Values are transformed — Wv transforms each position's vector before it's blended in. The model learns what to broadcast, not just that it broadcasts.

Analogy: a meeting where you assess each person's relevance to your question, weight their contribution accordingly, and each person only shares the part of their knowledge that's relevant.
Geometry and analysis — PCA, plots, and what they show
What each dot in the PCA plot represents

One dot = one sequence's final residual stream vector

500 test sequences × position 15 = 500 residual stream vectors, each 128D. PCA compresses 128D → 3D for visualization. Each dot is one sequence. Its position = where that sequence's belief state ended up in compressed space. Color = which Mess3 source generated it.

How many points: 500 (one per test sequence, using final position only). If all 16 positions were plotted: 500 × 16 = 8000 points.
PCA — what it does and why 3D

Finds directions of most variance for visualization

You have 500 points in 128D space — unplottable. PCA finds the 2 or 3 directions where the cloud is most spread out and projects every point onto those directions. PC1 captures the most variance (83.8%), PC2 the next most (7.0%), PC3 next (5.9%).

Why 3D vs 2D: purely a visualization choice. PC1 separates Source 0; PC2+PC3 reveal within-cluster structure. 3D shows more. The effective rank calculation uses all 128 dimensions regardless.
Effective rank and CEV

How many dimensions the model actually uses

Cumulative Explained Variance (CEV) at k = fraction of total variance in top k PCs. Effective rank = minimum k such that CEV(k) ≥ 95%. In your experiment: CEV(2) ≈ 96.7%, so effective rank = 3 (due to 0-indexing — you need PC1+PC2+PC3 to cross 95%).

Key test: does the model use 10 dims (factored, 5 factors × 2D each) or 242 dims (joint, 3⁵-1)? For your non-ergodic setup the predicted range was 2–8.
Layer vs. position analysis

Two independent axes for studying residual streams

You can study activations at any (layer, position) combination. Fixing the final layer and varying position shows how belief state geometry evolves as context accumulates — that's Figure 2 (effective rank by token position). Fixing a position and varying layer shows how representations develop through the network depth.

Effective rank trajectory — what 1→4→3 means

The model expands then contracts its representational space

Position 0 (BOS): rank 1 — all sequences identical, no variance. Positions 1–7: rank rises to 4 — active source disambiguation phase, model building up a representation to distinguish sources. Positions 8–15: rank drops to 3 — source identity resolved, model collapses onto identified source's belief simplex.

Intuition: the model uses extra dimensions while actively inferring, then discards them once inference converges. The rise then fall was not predicted — original prediction was monotonic decrease from 8 to 2.
Linear decodability (R²)

Can you read out belief states from the residual stream?

Linear regression from 128D residual stream → ground-truth belief states. High R² means the belief state is linearly encoded and readable. Results: hidden state beliefs R² = 0.83–0.97 (very high). Source weight R² = 0.37 (low — not because absent, but encoded categorically not continuously).

Why the asymmetry: hidden state directly determines next token → strong gradient pressure to encode linearly. Source identity is higher-level and influences predictions only indirectly.
The transformer forward pass — step by step

What happens when sequence [0, 1, 2, 3, 4] enters the transformer

Step 1 — Embedding
Each integer looked up in embedding table → 5 × 128D starting vectors. Each position knows only its own token.
Step 2 — Attention (layer 0)
All positions compute Q, K, V from their current vectors. Causal masking applied. Each position takes weighted sum of allowed positions' V vectors. Result added to residual stream.
Step 3 — MLP (layer 0)
Each position's updated vector passes through feedforward network independently. Result added to residual stream.
Step 4 — Repeat for layer 1
Same attention + MLP, now with richer residual stream vectors. Position 4 now has information from all prior positions through two rounds of attention.
Output
5 × 128D final residual stream vectors — one per position. Position 4's vector has seen everything. Position 0's vector has only ever seen itself.
